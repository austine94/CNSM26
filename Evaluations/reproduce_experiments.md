# Reproduce selected paper experiments

This file documents how to reproduce the requested subset of paper experiments from the `paper_experiments/` folder. It intentionally excludes the rest of the full experiment suite.

## Requirements to install before running

Install the project Python dependencies from the repository root:

```bash
pip install -r requirements.txt
```

Some selected experiments also require the following runtime dependencies and services:

- **Ollama Python package and a running Ollama service** for all model-calling sweeps.
- **Locally available Ollama models** used by these scripts, including:
  - `interstellarninja/llama3.1-8b-tools:latest`
  - `llama3.2:1b`
  - `llama3.2:latest`
  - `aratan/qwen3-4b-tools:latest`
  - `lukaspetrik/gemma3-tools:4b`
- **Gurobi / `gurobipy` and a valid licence** for experiments that solve allocation problems.
- **Existing upstream data for post-processing steps**:
  - `varying_prompt_regret_model_sweep_parameter_selection_accuracy.py` requires `experiment_data/varying_prompt_regret_model_sweep_results.json`, generated by `varying_prompt_regret_model_sweep.py`.
  - `plot_scalability_experiment.py` requires `experiment_data/scalability_experiment_results.json`, generated by `scalability_experiment.py`.
  - `parameter_accuracy_selection_std_tables.py` reads `experiment_data/parameter_accuracy_model_sweep_results.json` and `experiment_data/parameter_accuracy_temperature_sweep_results.json`. The model-sweep file is generated by this subset; the temperature-sweep file must already exist if you do not intend to run the full temperature-sweep experiment.
  - `conflict_parameter_selection_regret.py` requires `experiment_data/conflict_parameter_selection_results.json` unless you pass a different input file directly to that script.

## Run the selected subset with one command

From the repository root:

```bash
cd paper_experiments
python reproduce_experiments.py
```

The runner executes only the scripts listed below, in this order, and stops at the first failure by default. To print the commands without executing them, use:

```bash
python reproduce_experiments.py --dry-run
```

To keep running later scripts after a failure, use:

```bash
python reproduce_experiments.py --continue-on-error
```
